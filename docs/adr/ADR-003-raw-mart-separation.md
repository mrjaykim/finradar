# ADR-003: raw/mart 레이어 분리

## 상태
결정됨 (2026-07-03)

## 배경
DART API 응답을 그대로 저장할지, 정제된 형태로만 저장할지 결정 필요.

## 검토한 대안
1. 단일 레이어로 정제된 데이터만 저장
2. raw(원본 그대로)와 mart(정제됨) 레이어로 분리

## 선택
2번 채택.

## 선택 이유
- 정제 로직이 바뀌어도 원본 데이터를 다시 API로 수집할 필요 없이 raw에서 재생성(replay) 가능
- DART API 응답 필드가 결측되거나 예외적인 값을 가질 수 있어, 정제 없이 우선 안전하게 적재하는
  raw 레이어가 파이프라인 안정성을 높임
- company(차원)와 financial_statement(팩트)를 mart에서 분리해 SCD와 Fact 패턴을 명확히 함

## 트레이드오프
저장 공간이 늘어나고(원본+정제본 이중 저장), ETL 파이프라인에 변환 단계가 하나 더 필요해짐.
다만 데이터 정합성과 재처리 유연성 확보가 더 중요하다고 판단.
