# ADR-002: 데이터베이스로 PostgreSQL 선택

## 상태
결정됨 (2026-07-02)

## 배경
finradar는 DART 재무 데이터를 저장하고 분석 쿼리(재무비율 계산, 시계열 비교, 업종별 집계)를
수행할 RDBMS가 필요했음.

## 검토한 대안
- Oracle
- MySQL
- PostgreSQL

## 선택
PostgreSQL 채택.

## 선택 이유
- 완전 무료 오픈소스로 포트폴리오 프로젝트에 비용 부담 없음 (Oracle은 상용 라이선스)
- Window 함수, CTE 등 분석 쿼리에 강하고 JSON/JSONB 타입을 지원해 raw 레이어의 유연한 저장에 유리
- MySQL 대비 트랜잭션/정합성 처리가 엄격하고 복잡한 JOIN 최적화가 우수
- Apache Airflow의 공식 권장 메타데이터 백엔드 중 하나로, 향후 Airflow 도입 시 생태계가 일치함
- 최근 데이터 엔지니어링 채용 시장에서 요구 빈도가 높은 추세

## 트레이드오프
Oracle 경험을 요구하는 일부 레거시 금융권 채용 공고에는 정확히 대응하지 못함.
다만 포트폴리오 단계에서는 접근성과 학습 곡선을 우선함.
